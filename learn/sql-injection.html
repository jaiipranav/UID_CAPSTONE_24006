SQL Injection

SQL Injection (SQLi) is a code injection technique used to attack data-driven applications. Malicious SQL statements are inserted into an entry field for execution, allowing attackers to bypass authentication, retrieve sensitive data, modify database content, and even execute administrative operations.

How It Works

SQL injection vulnerabilities occur when an application uses user-provided data directly in an SQL query without proper sanitization. An attacker can supply specially crafted input that changes the structure of the SQL query, causing the database to execute the attacker's commands.

An Example Concept

Imagine a login form that queries the database like this: SELECT * FROM users WHERE username = 'USER_INPUT' AND password = 'PASS_INPUT';. An attacker could enter ' OR '1'='1 as the username. The query becomes SELECT * FROM users WHERE username = '' OR '1'='1' AND password = '...';. Since '1'='1' is always true, the database may return a list of users, bypassing the password check.

How to Prevent SQL Injection

  • Use Prepared Statements (Parameterized Queries): This is the most effective method. It separates the SQL query structure from the data, ensuring user input cannot change the query's intent.
  • Input Validation: Implement strict rules for what characters and formats are allowed in user input fields.
  • Principle of Least Privilege: The database user account that the application uses should only have the minimum permissions necessary.
